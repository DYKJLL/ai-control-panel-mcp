# 图片理解工具 — AI 使用指南

## 必读：使用顺序（必须按此执行）

### 第一步：查看当前目录
```
工具：get_vision_dir
参数：{}
```
AI 必须先调用这个，了解当前配置。

### 第二步：询问用户配置目录
AI 问用户："你想把图片存在哪个目录？"

用户回答后，调用：
```
工具：set_vision_dir
参数：{"path": "用户回答的路径", "account_id": "acc_577cd0d2"}
```

### 第三步：使用核心功能看图
目录配置完成后，才能用：
```
工具：browser_ask_vision
参数：{"image_source": "图片路径", "question": "问题", "account_id": "acc_577cd0d2"}
```

---

## 工具说明

### get_vision_dir
必用！查看当前存储目录。AI 首次使用必须先调用，然后询问用户想存哪里。

### set_vision_dir
必用！AI 先问用户，然后把用户回答的路径传给这个工具配置。

### browser_ask_vision
核心功能。上传图片到豆包，让豆包描述图片，返回文字回答。

---

## 常见错误处理

| 错误 | 做法 |
|------|------|
| `找不到 + 按钮` | 等2秒重试 |
| `上传面板未打开` | 等2秒重试 |
| `未在30秒内获取到AI回复` | 等5秒重试 |
| `browser not open` | 检查服务是否运行 |
| 目录没配置 | 先调用 get_vision_dir 再 set_vision_dir |

---

## 图片来源格式

- 本地路径：`C:\Users\xxx\photo.png`（绝对路径）
- 网络 URL：`https://example.com/image.png`
- Base64：`data:image/png;base64,...`

---

## 注意事项

- 图片会暂存本地目录（用户配置的目录）
- 图片会上传到豆包服务器，**不要传隐私图片**
- 每次调用约 10-20 秒
- 图片路径必须是**绝对路径**，不能是相对路径

---

## 环境状态

- HTTP 服务：`http://127.0.0.1:1984`（main.py）
- MCP 服务：stdio 管道（mcp_server.py）

两个服务均已在后台运行，AI 可直接调用。
